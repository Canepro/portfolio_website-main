# Roadmap

This document is a lightweight, living roadmap for the portfolio website.

## Current focus (next PRs)

- Keep the **Featured Projects** section accurate, recruiter-friendly, and backed by real links.
- Improve **documentation clarity** (reduce duplication; keep one “source of truth” per topic).
- Keep the repo clean: remove stale assets, keep root tidy, and avoid broken references.

## Near-term enhancements

- **Portfolio Metrics → Grafana**: continue tightening the “portfolio monitors itself” demo.
- **Project media polish**: consistent aspect ratio thumbnails; optimize large PNG/GIF sizes.
- **UX polish**: mobile nav behavior, social preview validation, and sitemap submission.

## Longer-term ideas

- **AI-powered DevOps assistant** (log/metric query + suggestions).
- **Multi-cloud GitOps** expansion (additional clusters/providers).
- **Zero-trust hardening** patterns (policy-as-code, mTLS, OIDC).

## How to propose a new project

When adding a new project, provide:

- Title, slug, category, short description
- Tags, code link, live demo link (if any)
- Long description (Markdown), challenges/solutions, impact, technologies
- A thumbnail image in `public/images/`
